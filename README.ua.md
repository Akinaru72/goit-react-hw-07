# Домашнє завдання — goit-react-hw-07

## Загальні вимоги

- Створіть репозиторій з назвою **goit-react-hw-07**
- Під час здачі домашньої роботи надайте **два посилання**:
  - на репозиторій з вихідними файлами
  - на робочу сторінку завдання, задеплоєну на [**Vercel**](https://vercel.com/)
- Проєкт повинен бути створений за допомогою [**Vite**](https://vite.dev/)
- Під час запуску коду в консолі **не повинно бути помилок або попереджень**
- Для кожного компонента у папці `src/components` має бути **окрема папка**, яка містить:
  - JSX-файл React-компонента
  - файл стилів
- Назва папки, файла компонента (з розширенням `.jsx`) та файла стилів (перед `.module.css`) повинні бути **однаковими** та відповідати назвам, вказаним у завданні (якщо вони були)
- Для експорту компонентів використовується **експорт за замовчуванням** (`export default`)
- JS-код має бути чистим і зрозумілим — використовуйте **Prettier**
- Використовується бібліотека **Redux Toolkit**
- Стилізація повинна бути виконана за допомогою **CSS-модулів**

## Форма стану

Додайте в Redux-стан обробку:

- індикатора завантаження
- помилок HTTP-запитів

Для цього змініть форму стану **слайсу контактів**, додавши властивості `loading` та `error`.

### Приклад структури стану:

```js
{
  contacts: {
    items: [],
    loading: false,
    error: null
  },
  filters: {
    name: ""
  }
}
```

## Операції

У папці `redux` створи файл `contactsOps.js` для зберігання асинхронних генераторів екшенів.

### Вимоги

- Для оголошення операцій використовуй функцію [`createAsyncThunk`](https://redux-toolkit.js.org/api/createAsyncThunk)
- Для виконання HTTP-запитів використовуй бібліотеку [**axios**](https://axios-http.com/uk/docs/intro)

### Необхідно оголосити такі операції:

- **fetchContacts** — отримання масиву контактів
  - HTTP-метод: `GET`
  - Базовий тип екшену: `"contacts/fetchAll"`

- **addContact** — додавання нового контакту
  - HTTP-метод: `POST`
  - Базовий тип екшену: `"contacts/addContact"`

- **deleteContact** — видалення контакту за `id`
  - HTTP-метод: `DELETE`
  - Базовий тип екшену: `"contacts/deleteContact"`

### Обробка помилок

Для коректного опрацювання HTTP-помилок всередині операцій:

- Використовуй конструкцію `try...catch`
- У блоці `catch` повертай результат виклику  
  `thunkAPI.rejectWithValue(...)`

## Функції-селектори

### Слайс контактів

У файлі `contactsSlice.js` оголоси та експортуй такі селектори для використання з `useSelector`:

- **selectContacts** — повертає список контактів з `items`
- **selectLoading** — повертає стан завантаження `loading`
- **selectError** — повертає значення `error`

Використовуй ці селектори у всіх компонентах, де потрібен доступ до відповідних властивостей стану.

---

### Слайс фільтра

У файлі `filtersSlice.js` оголоси та експортуй селектор:

- **selectNameFilter** — повертає значення фільтра з властивості `name`

Використовуй цей селектор у компонентах разом із хуком `useSelector`.

---

## Мемоізація селекторів

Після додавання властивостей `loading` та `error` у слайс контактів виникає проблема оптимізації, оскільки фільтрація контактів буде виконуватись не лише при зміні контактів або фільтра, а й при зміні `loading` чи `error`.

### Для вирішення:

- У файлі `contactsSlice.js` створи та експортуй мемоізований селектор  
  **selectFilteredContacts** за допомогою функції [`createSelector`](https://redux-toolkit.js.org/api/createSelector)
- Селектор повинен залежати від:
  - масиву контактів
  - значення фільтра
- Селектор має повертати відфільтрований масив контактів
- Імпортуй `selectFilteredContacts` у компонент **ContactList.jsx**
  та використовуй його в `useSelector`
